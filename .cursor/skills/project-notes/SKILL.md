# 项目笔记记录技能

## 工作原则

**笔记是一棵树：**
- **主干** = 模块的基础知识介绍（功能、作用、在架构中的位置）
- **分支** = 子模块的功能说明
- **叶子** = 具体 Q&A（每个问题记录时都要标注所属模块）

每次用户问完问题后说"记录问题"，就将 Q&A 归入对应模块。

## 笔记文件

- 主笔记：`references/notes.md` — Q&A 叶子，按模块组织
- 项目树：`references/project_tree.md` — 完整模块树 + 各模块基础知识（主干）
- 进度：`references/progress.md` — 已记录 Q&A 数量、已覆盖模块

## 记录流程

### 第一步：识别用户问题属于哪个模块

读取 `references/project_tree.md`，根据问题内容匹配模块。

如果问题涉及多个模块，在每个模块下都记录。

如果问题落在已有模块但该模块还没有主干介绍，**先补充主干再记录叶子**。

### 第二步：记录格式

**叶子格式（所有模块通用）：**

```markdown
### Q: [问题原文]

**A:** [完整回答]

- 涉及文件：`filepath`（可选）
- 记录时间：`YYYY-MM-DD`
```

**主干格式（首次为某模块记录时补充）：**

```markdown
## [D?] [模块中文名]

**文件位置：** `module/path/`
**作用：** 1-2 句话描述核心功能
**在架构中的位置：** 1-2 句话描述与其他模块的关系
**核心组件：**
- `component.py` — 作用说明
```

### 第三步：更新进度

每次记录后更新 `references/progress.md`，标注：
- 总 Q&A 数
- 新覆盖的模块
- 待完善的模块

## 注意事项

- 记录的是**你自己给出的回答**，不是用户原话复述
- 如果同一问题在不同时间被重复问到，只在叶子下追加新的补充说明，不重复记录
- 对于项目全局性问题（如"整体架构"），归入 **D0: 架构总览**
- 每次记录前先读一下 `references/notes.md` 的末尾，确认格式一致
